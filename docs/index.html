GitHub 回復バッジ

フリーメール等が原因のアカウント制限（サードパーティログイン拒否・シャドウバン）から 回復したことを、GitHub ログインで本人確認しながら証明するバッジ。 フォークもトークンも不要です。

STEP 1 — 登録（GitHub ログインが必要）

ボタンを押して Issue を送るだけ

Issue の作成には GitHub ログインが必須です。採番 Bot は、あなたが入力した文字ではなく GitHub が検証した本人のログイン名だけを使って登録します。 だから他人になりすまして番号を取ることはできません。

  1. 下のボタンで登録 Issue を開く（ログイン状態で送信）
  2. Bot が数十秒であなた専用のバッジ Markdown と通し番号をコメント
  3. Bot は登録前に github.com/あなた が実際に見える状態か（＝本当に回復済みか）を確認します
GitHub で登録する →

STEP 2 — 貼る

Bot が返したバッジを README やブログへ

渡されるバッジは shields.io が github.com/あなた を直接見て判定します。 トークン不要・自己申告なしで、閲覧するたびにリアルタイム検証。制限が再発すれば 自動で赤 restricted に戻り、回復していれば緑 unrestricted。
